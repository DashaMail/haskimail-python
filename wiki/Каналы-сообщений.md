# Каналы сообщений (Message Streams)

Каналы позволяют разделять типы рассылок (транзакционные, маркетинговые) на одном сервере.

## Список каналов

```python
from haskimail import ServerClient

client = ServerClient("ваш-серверный-токен")

streams = client.get_message_streams(
    MessageStreamType="All",
    IncludeArchivedStreams=False,
)

for s in streams["MessageStreams"]:
    print(f"[{s['ID']}] {s['Name']} — {s['MessageStreamType']}")
```

### Параметры фильтрации

| Параметр | Тип | Описание |
|----------|-----|----------|
| `MessageStreamType` | string | `"All"`, `"Transactional"`, `"Broadcasts"` |
| `IncludeArchivedStreams` | bool | Включать архивные каналы |

## Получить канал

```python
stream = client.get_message_stream("outbound")
print(f"Имя: {stream['Name']}")
print(f"Тип: {stream['MessageStreamType']}")
```

## Создать канал

```python
stream = client.create_message_stream({
    "ID": "notifications",
    "Name": "Уведомления",
    "Description": "Канал для уведомлений о событиях",
    "MessageStreamType": "Transactional",
})

print(f"Создан канал: {stream['ID']}")
```

### Ограничения

- Макс. 10 каналов на сервер
- ID: только строчные латинские буквы, цифры, `-`, `_` (до 30 символов)
- ID не может начинаться с `hm-`
- ID `all` зарезервирован

## Редактировать канал

```python
client.edit_message_stream("notifications", {
    "Name": "Важные уведомления",
    "Description": "Обновлённое описание",
})
```

## Архивировать канал

```python
client.archive_message_stream("notifications")
```

> Стандартный канал `outbound` архивировать нельзя.

## Разархивировать канал

```python
client.unarchive_message_stream("notifications")
```
